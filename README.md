Для всех алгоритмов наблюдается ожидаемый рост времени выполнения при увеличении размера массива.
Также можно наблюдать, что Gybrid sort показывает результаты лучше, чем Merge sort при любом типе массива.

Почти отсортированные массивы (NearlySorted):
Лучше всего себя показывает Gybrid c threshold = 80, но несильно отстает threshold = 40.
Merge работает медленнее из-за дополнительных накладных расходов на рекурсию и слияние.

Случайные массивы (Random):
Gybrid c threshold = 40 стабильно демонстрирует наилучшее время, но при threshold = 10 и threshold = 80 результаты ухудшаются незначительно.
Merge работает медленнее из-за дополнительных накладных расходов на рекурсию и слияние.

Обратно отсортированные массивы (Reversed):
Gybrid c threshold = 40 и threshold = 10 показывают примерно одинаковые результаты, но при threshold = 80 демонстрация времени не лучшая, но все же быстрее чем Merge sort. Merge работает медленнее из-за дополнительных накладных расходов на рекурсию и слияние.

В итоге, в среднем лучше всего себя показывает Gybrid c threshold = 40. На каждом типе массива он демонстрирует хорошие показатели.
